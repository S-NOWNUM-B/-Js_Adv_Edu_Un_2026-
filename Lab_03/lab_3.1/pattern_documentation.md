# Формальное описание паттерна: Validation Pattern

Pattern Name: Validator Pattern

Description: Паттерн для создания многоразовых правил проверки данных, отделяющих логику валидации от бизнес-логики

Context Outline: Любая часть приложения, принимающая ввод от пользователя или внешнего источника.

Problem Statement: Разрозненная проверка данных ведет к дублированию кода и риску пропустить важные проверки безопасности

Solution: Создание набора функций-валидаторов, которые последовательно проверяют данные на соответствие правилам

Design: Реализуется как утилитарный класс или набор чистых функций, принимающих значение и возвращающих результат

Implementation: Описание логики через регулярные выражения и условные операторы

Consequences:
- Преимущества: Чистый код, легкое добавление новых правил, централизованная безопасность.
- Недостатки: Незначительное увеличение объема кода для простых проверок.
