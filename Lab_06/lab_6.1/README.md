# Lab 6.1: Pattern Classification and Identification

## Обзор
Лабораторная работа посвящена изучению классификации паттернов проектирования Gang of Four (GoF), идентификации паттернов в фрагментах кода и анализу важности правильной категоризации для архитектурных решений.

## Цель работы
Научиться распознавать классические паттерны проектирования в коде, правильно относить их к трём семействам (Creational, Structural, Behavioral) и понимать различие между Class-based и Object-based подходами.

## Структура файлов в lab_6.1:
*   `classification.md` — Классификация 10 сниппетов кода по семействам паттернов GoF и идентификация конкретных паттернов с обоснованием выбора.
*   `gof_mapping.md` — Сопоставление идентифицированных паттернов с уровнями Class vs Object согласно классификации GoF.
*   `reflection.md` — Рефлексия о важности категоризации паттернов для избежания архитектурных ошибок и антипаттернов.

## Идентифицированные паттерны:
1.  **Singleton** (Creational, Object) — Единственный экземпляр класса
2.  **Observer/Pub-Sub** (Behavioral, Object) — Подписка на события
3.  **Facade** (Structural, Object) — Упрощённый интерфейс подсистем
4.  **Factory Method** (Creational, Class) — Делегирование создания объектов
5.  **Decorator** (Structural, Object) — Динамическое добавление функциональности
6.  **Prototype** (Creational, Object) — Клонирование объектов
7.  **Chain of Responsibility** (Behavioral, Object) — Передача запроса по цепочке
8.  **Command** (Behavioral, Object) — Инкапсуляция действий в объекты
9.  **Flyweight** (Structural, Object) — Оптимизация памяти через разделение состояния
10. **Mediator** (Behavioral, Object) — Централизация взаимодействия объектов

## Методология
Анализ выполнен через изучение структуры кода, выявление ключевых признаков паттернов (методы, свойства, композиция), сопоставление с описаниями из книги GoF и валидация выбора через понимание решаемой проблемы.
