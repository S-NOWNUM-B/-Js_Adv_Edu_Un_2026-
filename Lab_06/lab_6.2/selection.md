# Scenario Selection Records

### SCENARIO_01
- **Problem:** Нужно создать единое хранилище конфигурации для всего приложения, чтобы избежать дублирования данных.
- **Pattern:** Singleton (Creational)
- **Why Family:** Проблема в *создании* объекта — нам нужно гарантировать, что существует ровно один экземпляр.
- **Why Pattern:** Singleton обеспечивает глобальную точку доступа к настройкам и предотвращает создание копий.

### SCENARIO_02
- **Problem:** Старый метод оплаты `processPayment` не совпадает по названию и аргументам с новым интерфейсом `charge`.
- **Pattern:** Adapter (Structural)
- **Why Family:** Изменяется *структура* взаимодействия между несовместимыми интерфейсами.
- **Why Pattern:** Adapter оборачивает старый интерфейс, приводя его к виду, который ожидает современный код.

### SCENARIO_03
- **Problem:** Множество компонентов должны узнавать о сдаче домашнего задания, чтобы обновить свои данные.
- **Pattern:** Observer (Behavioral)
- **Why Family:** Речь идет о *коммуникации* между независимыми объектами.
- **Why Pattern:** Позволяет компонентам подписываться на события и реагировать на них без жесткой привязки.

### SCENARIO_04
- **Problem:** Сложный процесс создания отчета (Header, Body, Footer) в разных форматах (HTML/PDF).
- **Pattern:** Builder (Creational)
- **Why Family:** Контроль процесса *создания* сложного объекта.
- **Why Pattern:** Позволяет использовать один и тот же процесс конструирования для создания различных представлений отчета.

### SCENARIO_05
- **Problem:** Система управления задачами требует функций отмены (Undo) и повтора (Redo).
- **Pattern:** Command (Behavioral)
- **Why Family:** Абстрагирование *действий* от объектов, которые их вызывают.
- **Why Pattern:** Каждое действие сохраняется как объект с методами `execute` и `undo`, что позволяет управлять историей.

### SCENARIO_06
- **Problem:** Приложение падает из-за нехватки памяти при создании 50 000 объектов частиц.
- **Pattern:** Flyweight (Structural)
- **Why Family:** Оптимизация *композиции* объектов для экономии памяти.
- **Why Pattern:** Позволяет вынести общие данные частиц (цвет, текстура) в разделяемые объекты, экономя мегабайты памяти.
