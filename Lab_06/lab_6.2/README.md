# Lab 6.2: Pattern Selection and Implementation

## Обзор
В данной лабораторной работе выполнен анализ 6 практических сценариев с типовыми проблемами в разработке, выбор подходящих паттернов проектирования для их решения и реализация двух выбранных сценариев на JavaScript.

## Цель работы
Научиться анализировать контекст проблемы, определять подходящее семейство паттернов, выбирать конкретный паттерн и реализовывать его для решения реальных архитектурных задач.

## Структура файлов в lab_6.2:
*   `selection.md` — Обоснование выбора паттерна для каждого из 6 сценариев с указанием семейства, конкретного паттерна и аргументацией выбора.
*   `decision_journal.md` — Журнал принятия решений: какие сценарии реализованы, ключевые улики для остальных и выводы о применении паттернов.
*   `src/impl_scenario_01.js` — Реализация паттерна **Singleton** для системы конфигурации приложения.
*   `src/impl_scenario_02.js` — Реализация паттерна **Adapter** для интеграции legacy API оплаты с новым интерфейсом.

## Проанализированные сценарии:
1.  **Singleton** (Creational) — Глобальное хранилище конфигурации без дублирования данных *реализовано*
2.  **Adapter** (Structural) — Совместимость старого API `processPayment` с новым интерфейсом `charge` *реализовано*
3.  **Observer** (Behavioral) — Система уведомлений о сдаче домашних заданий для независимых компонентов
4.  **Builder** (Creational) — Генератор отчётов с пошаговым созданием в форматах HTML/PDF
5.  **Command** (Behavioral) — Система Undo/Redo для управления задачами
6.  **Flyweight** (Structural) — Оптимизация памяти для 50,000 частиц через разделение состояния

## Методология
Работа выполнена в три этапа:
1.  **Анализ проблемы:** Идентификация ключевых слов и контекста каждого сценария.
2.  **Выбор паттерна:** Определение подходящего семейства (Creational/Structural/Behavioral) и конкретного паттерна.
3.  **Реализация:** Написание кода для двух сценариев с демонстрацией работы паттернов.

## Запуск реализаций:

# Adapter — Интеграция legacy API
node Lab_06/lab_6.2/src/impl_scenario_02.js
