# Lab 8.1: Flyweight Pattern

## Обзор
В данной части лабораторной работы реализован структурный паттерн **Flyweight** для массового рендеринга деревьев на Canvas с повторным использованием общего состояния.

## Цель работы
Понять, как отделять внутреннее состояние объектов от внешнего и снижать потребление памяти при создании большого количества однотипных сущностей.

## Структура файлов в lab_8.1:
*   `tree/FlyweightTree.js` — Легковес с внутренним состоянием (тип, вид, цвета, размер) и методом `render`.
*   `tree/TreeFactory.js` — Фабрика легковесов с кешированием по ключу параметров и сбором статистики.
*   `forest/ForestRender.js` — Генератор леса и рендерер экземпляров с внешним состоянием (позиция, масштаб, поворот).
*   `index.html` — Точка запуска демонстрации (генерация 10 000 деревьев и вывод статистики).

## Ключевые компоненты

### FlyweightTree
- Хранит только внутреннее (разделяемое) состояние дерева
- Отрисовывает дерево на Canvas с учетом внешних параметров
- Используется многократно для множества экземпляров

### TreeFactory
- Возвращает существующий легковес или создает новый
- Кеширует легковесы в `Map` по составному ключу
- Считает общее число деревьев и количество уникальных легковесов

### ForestRenderer
- Генерирует массив деревьев с координатами и трансформациями
- Использует фабрику для получения разделяемых объектов
- Рендерит все деревья и возвращает статистику экономии памяти

## Архитектурные преимущества
- Значительное уменьшение количества реально созданных объектов
- Экономия памяти при работе с большими коллекциями сущностей
- Простое расширение новыми типами деревьев без изменения базовой логики
