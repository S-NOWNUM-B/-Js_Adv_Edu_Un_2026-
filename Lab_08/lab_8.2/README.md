# Lab 8.2: Event Delegation Pattern

## Обзор
В данной части лабораторной работы реализован подход **Event Delegation** для управления большим списком задач через минимальное число обработчиков событий.

## Цель работы
Освоить делегирование DOM-событий на родительский контейнер и понять, как это улучшает производительность и упрощает сопровождение интерфейса.

## Структура файлов в lab_8.2:
*   `delegate/EventDelegator.js` — Делегатор событий с едиными слушателями `click` и `dblclick`.
*   `todo/TodoList.js` — Логика TodoList: создание задач, переключение статуса, удаление и изменение приоритета.
*   `index.html` — Демонстрация списка из 1000 элементов и проверка числа слушателей в DevTools.

## Ключевые компоненты

### EventDelegator
- Устанавливает обработчики на контейнер, а не на каждый элемент
- Определяет действие через `data-action`
- Передает событие в пользовательские обработчики через `emit`

### TodoList
- Хранит задачи в `Map` и генерирует DOM-элементы
- Подписывается на события делегатора (`delete`, `toggle`, `priority`)
- Обновляет состояние и DOM без создания множества отдельных слушателей

## Архитектурные преимущества
- Константное число обработчиков при росте количества элементов
- Снижение нагрузки на память и упрощение управления событиями
- Удобное масштабирование интерфейса при динамическом добавлении задач
