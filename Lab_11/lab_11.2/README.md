# Lab 11.2: Advanced Async Patterns

## Обзор
В данной части лабораторной работы реализованы продвинутые асинхронные паттерны: асинхронный pipeline для обработки данных слоями, support для отмены операций через AbortController, и различные стратегии обработки ошибок при выборке данных.

## Цель работы
Освоить продвинутые асинхронные паттерны для построения гибких и масштабируемых сторон обработки данных с поддержкой отмены и fallback механизмов.

## Структура файлов в lab_11.2:
* `async/Pipeline.js` — Асинхронный pipeline для последовательной обработки данных через middleware слои и утилиты для параллельной обработки массивов
* `async/CancellableFetcher.js` — HTTP запросы с поддержкой отмены через AbortController
* `async/DataFetcher.js` — Выборка данных с использованием `Promise.all`, `Promise.allSettled` и fallback механизмов
* `index.html` — Демонстрация работы асинхронных операций в браузере

## Ключевые компоненты

### AsyncPipeline
- Обработка данных через последовательность middleware функций
- Каждый слой может трансформировать данные для следующего слоя
- Раннее прерывание при получении null/undefined
- Утилита `mapParallel()` для параллельной обработки массивов с контролем одновременности
- `AsyncBatcher` для группировки элементов и обработки пакетами

### CancellableFetcher
- Отмена предыдущих запросов при новом запросе по тому же ID
- Использование `AbortController` для четкой отмены fetch операций
- `withCancellation()` функция для комбинирования Promise с сигналом отмены

### DataFetcher
- `fetchUserWithDetails()` — параллельное получение пользователя, постов и подписчиков через `Promise.all`
- `fetchMultipleUsers()` — использование `Promise.allSettled` для получения положительных и отрицательных результатов
- `fetchWithFallback()` — последовательный перебор альтернативных URL до первого успеха
- Error handling и логирование для каждого сценария

## Архитектурные преимущества
- Гибкая обработка данных через pipeline слои
- Поддержка отмены долгоживущих операций
- Graceful degradation через fallback механизмы
- Одновременная обработка неизвестного количества элементов с контролем
- Разделение критерия успеха и неудачи через `allSettled`
