# AI_report lab_11.1

## 1. Информация о использовании AI
- Инструмент: Copilot / GitHub, Gemini / Google AI Studio
- Дата: 26.03.2026

## 2. Конкретные промпты

### Промпт 1:
"Как реализовать автоматический retry для Promise с экспоненциальной задержкой (exponential backoff)"

Что получил: Алгоритм с начальной задержкой, умножением на коэффициент при каждой попытке и максимальной задержкой. Примеры retryableStatuses и retryableErrors.

### Промпт 2:
"Как ограничить время выполнения Promise через timeout"

Что получил: Паттерн с `Promise.race()` между основным Promise и timer-ом, который отклоняет Promise после таймаута.

### Промпт 3:
"Как реализовать управление одновременностью асинхронных операций (concurrency control)"

Что получил: Структару AsyncQueue с поддержкой N одновременных операций, хранением в очереди оставшихся задач и обработкой по мере освобождения слотов.

### Промпт 4:
"Как ограничить частоту запросов (rate limiting) в системе API"

Что получил: RateLimiter с отслеживанием времени запросов, фильтрацией старых записей и расчетом времени ожидания для соблюдения лимита.

### Промпт 5:
"Как организовать HTTP клиент с обработкой ошибок и стандартными заголовками"

Что получил: Кастомный класс HttpError с информацией о статусе, базовый URL в конструкторе, проверка response.ok и распознавание JSON.

## 3. Модификации и проверки
- Реализован `PromiseHttpClient` с обработкой ошибок и поддержкой GET/POST/PUT/DELETE методов
- Создан `DataFetcher` с использованием retry и timeout утилит
- Реализована `AsyncQueue` с контролем одновременности N операций
- Создан `RateLimiter` для ограничения частоты запросов
- Реализованы утилиты `withRetry()`, `withTimeout()` и `withRetryAndTimeout()`

## 4. Что узнал
1. Как реализовать экспоненциальный backoff для retry стратегии
2. Как использовать Promise.race для реализации timeout
3. Как контролировать одновременность асинхронных операций в очереди
4. Как реализовать rate limiting на основе времени
5. Как комбинировать несколько асинхронных паттернов для надежности

## 5. Процент использования
10 - 15% (AI помог только с идеями/объяснениями)

# AI_report lab_11.2

## 1. Информация о использовании AI
- Инструмент: Copilot / GitHub, Gemini / Google AI Studio
- Дата: 26.03.2026

## 2. Конкретные промпты

### Промпт 1:
"Как реализовать асинхронный pipeline для обработки данных через несколько слоев middleware"

Что получил: Паттерн с последовательным прохождением данных через функции, где каждая преобразует результат для следующей, с поддержкой контекста.

### Промпт 2:
"Как отменить fetch запрос в процессе выполнения"

Что получил: Использование `AbortController` с передачей сигнала в fetch options, вызов `abort()` для отмены запроса.

### Промпт 3:
"Как комбинировать Promise.all и Promise.allSettled в зависимости от требований"

Что получил: `Promise.all` для критичных операций (fail-fast), `Promise.allSettled` для независимых операций где нужны все результаты.

### Промпт 4:
"Как реализовать fallback механизм для попытки альтернативных источников данных"

Что получил: Цикл по массиву URL с try-catch, возврат первого успешного результата или ошибка если все неудачны.

### Промпт 5:
"Как параллельно обработать массив элементов с контролем одновременности"

Что получил: Разделение массива на чанки размером N и использование Promise.all для параллельной обработки каждого чанка последовательно.

### Промпт 6:
"Как реализовать AsyncBatcher для группировки элементов и обработки пакетами"

Что получил: Накопление элементов в очередь с автоматической отправкой при достижении размера бэтча или таймауте.

## 3. Модификации и проверки
- Реализован `AsyncPipeline` с поддержкой middleware функций и контекста
- Создан `CancellableFetcher` с поддержкой отмены через AbortController
- Реализован `DataFetcher` с Promise.all, Promise.allSettled и fallback сценариями
- Добавлена утилита `mapParallel()` для параллельной обработки с контролем одновременности
- Реализован `AsyncBatcher` для пакетной обработки элементов

## 4. Что узнал
1. Как строить гибкие pipeline архитектуры для обработки данных
2. Как использовать AbortController для отмены fetch операций
3. Разницу между Promise.all (fail-fast) и Promise.allSettled (all results)
4. Как реализовать graceful degradation через fallback механизмы
5. Как контролировать одновременность при параллельной обработке массивов
6. Как группировать операции в батчи для эффективной обработки

## 5. Процент использования
10 - 15% (AI помог только с идеями/объяснениями)
