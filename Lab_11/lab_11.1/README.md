# Lab 11.1: Promise-based Async Patterns

## Обзор
В данной части лабораторной работы реализованы асинхронные паттерны на основе Promises для надежной работы с сетевыми запросами: автоматический retry с экспоненциальной задержкой, управление timeout и контроль одновременности запросов.

## Цель работы
Освоить асинхронные паттерны JavaScript для обработки ошибок при работе с API, гарантирующие надежность и эффективность сетевого взаимодействия.

## Структура файлов в lab_11.1:
* `async/PromiseHttpClient.js` — HTTP клиент на базе Fetch API с обработкой ошибок и стандартными заголовками
* `async/DataFetcher.js` — Адаптер данных с методами fetch для пользователей и постов с автоматическим retry и timeout
* `async/AsyncQueue.js` — Управление очередью асинхронных операций с контролем одновременности и RateLimiter для ограничения частоты запросов
* `async/retry.js` — Утилиты для retry с экспоненциальной задержкой и timeout функциональность
* `index.html` — Демонстрация работы fetch операций в браузере

## Ключевые компоненты

### PromiseHttpClient
- Обертка над Fetch API с обработкой ошибок
- Конфигурация base URL и стандартных заголовков
- Распознавание JSON и текстовых ответов
- Кастомный класс `HttpError` с информацией о статусе и URL

### DataFetcher
- Получение пользователя по ID с автоматическим retry
- Получение постов по ID пользователя
- Параллельное получение пользователя и его постов через `Promise.all`
- Конфигурируемые timeout и maxAttempts

### AsyncQueue и RateLimiter
- `AsyncQueue` — управление одновременностью: выполнение N операций одновременно
- `RateLimiter` — ограничение частоты запросов: максимум N запросов в временном окне

### Retry утилиты
- `withRetry()` — автоматический retry с экспоненциальной задержкой (backoff)
- `withTimeout()` — ограничение времени выполнения Promise
- `withRetryAndTimeout()` — комбинация retry и timeout в одну операцию
- Конфигурируемые параметры: maxAttempts, baseDelay, maxDelay, retryableStatuses

## Архитектурные преимущества
- Надежность при неустойчивых сетевых соединениях
- Контроль одновременности для предотвращения перегруза сервера
- Ограничение частоты запросов через rate limiting
- Гибкая конфигурация timeout и retry стратегии
- Разделение HTTP клиента и бизнес-логики fetch
