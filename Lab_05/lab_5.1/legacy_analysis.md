# Анализ Legacy кода

### Антипатеные и устаревшие конструкции кода:
1) Использование var: Переменные taxRate, current, total, cart, p1 объявлены через var, что создаёт риск поднятия и загрязнения глобальной области видимости
2) Глобальная область видимости: Весь код находится в одном файле без инкапсуляции. Перепенные taxRate и current доступны отовсюду
3) Функции-конструкции: Product и Cart реализованы через функции и prototypeб что менее читаемо, чем современные классы
4) Отсутствие приватности: Все свойства явялются публичными. Любой код может напрямую изменить массив корзины
5) Конкатенация строк: Используется оператор + для формирования строк вместо шаблонных литералов
6) Function Declarations: Использубтся обычные функции там, где стрелочные функции были бы лаконичнее
