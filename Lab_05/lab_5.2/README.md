# Lab 5.2: Dynamic Module Loading and Lazy Components

## Обзор
Этап второй лабораторной работы № 5 посвящен внедрению передовых техник работы с модулями: динамический импорт (dynamic import), ленивая загрузка компонентов по событиям и видимости, оптимизация загрузки при инициализации приложения.

## Цель работы
Освоить динамический импорт модулей, применить паттерны lazy loading для оптимизации производительности веб-приложения, и научиться управлять асинхронной загрузкой компонентов.

## Структура проекта

### Модули в `modules/`:

- **`app.mjs`** — Главный модуль приложения:
  - Инициализация UI элементов
  - Event listeners для кнопок
  - IntersectionObserver для загрузки при видимости в viewport

- **`core.mjs`** — Ядро функциональности:
  - `getAppInfo()` — Получение информации о приложении
  - `getStatus()` — Получение статуса системы

- **`constants.mjs`** — Константы конфигурации:
  - `APP_CONFIG` — Объект с параметрами приложения (title, version, apiStatus, currency)

- **`utils.mjs`** — Утилиты:
  - `formatCurrency()` — Форматирование денежных значений
  - `modLog()` — Логирование с префиксом модуля

- **`advanced_feature.mjs`** — Продвинутая функция (загружается динамически):
  - `renderStats()` — Отрисовка статистики при нажатии кнопки

- **`lazy_component.mjs`** — Компонент отзывов (загружается при скроллинге):
  - `renderReviews()` — Отрисовка компонента отзывов в viewport

- **`index.html`** — Главный HTML файл:
  - Tailwind CSS для стилизации
  - Semantic разметка с id-селекторами для JS
  - Точка подключения основного модуля

## Ключевые техники

### Dynamic Import (Динамический импорт)
```javascript
// Загрузка модуля только при необходимости
const { renderStats } = await import('./advanced_feature.mjs');
```

**Преимущества:**
- Модули загружаются только когда они нужны
- Уменьшение размера initial bundle
- Возможность обработки ошибок загрузки

### Lazy Loading по событиям
```javascript
statsBtn.addEventListener('click', async () => {
  const { renderStats } = await import('./advanced_feature.mjs');
  renderStats('#stats-container', formatted);
  statsBtn.remove();
});
```

**Сценарий:** Загрузка аналитики только при клике пользователя

### IntersectionObserver для загрузки по видимости
```javascript
const observer = new IntersectionObserver(async (entries) => {
  if (entries[0].isIntersecting) {
    const module = await import('./lazy_component.mjs');
    module.renderReviews('#reviews-container');
  }
}, { threshold: 0.5 });

observer.observe(reviewsSection);
```

**Сценарий:** Загрузка отзывов только когда пользователь прокручивает до них

### Error Handling для асинхронных модулей
```javascript
try {
  const module = await import('./advanced_feature.mjs');
  // use module
} catch (err) {
  console.error('Failed to load module:', err);
}
```

## Архитектурные преимущества

### Separation of Concerns
- Каждый модуль отвечает за одну функцию
- Четкие границы между компонентами

### Code Splitting
- Initial bundle содержит только необходимый код
- Дополнительные функции загружаются по требованию

### Performance Optimization
- Меньше кода на первоначальной загрузке страницы
- Асинхронная загрузка не блокирует UI
- Lazy loading по видимости экономит ресурсы

### Maintainability
- Легче добавлять новые функции (новые .mjs файлы)
- Проще тестировать отдельные модули
- Структура понятна и масштабируема
