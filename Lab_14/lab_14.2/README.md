# Lab 14.2: Islands Architecture на Vanilla JavaScript

## Обзор
В данной лабораторной работе реализован пример **Islands Architecture**: большая часть страницы остаётся статическим HTML, а интерактивность подключается только для отдельных «островов». Это снижает объём исполняемого JavaScript и улучшает производительность.

## Цель работы
Освоить подход islands: выделение интерактивных зон, регистрация их поведения и ленивая гидрация только при необходимости.

## Структура файлов
- `index.html` — статическая страница с двумя интерактивными островами
- `island-engine.js` — реестр island-поведений и механизм отложенной гидрации

## Ключевые элементы реализации

### Статический океан + интерактивные острова
- Основной контент страницы является статическим и доступен без JavaScript
- Интерактивные блоки размечены атрибутом `data-island-type`
- Реализованы острова: `counter` и `color-picker`

### Реестр островов (IslandRegistry)
- `counter` управляет увеличением/уменьшением количества единиц
- `color-picker` изменяет цвет блока предпросмотра по кнопкам палитры
- Каждый остров инициализируется изолированно внутри своего контейнера

### Discovery и lazy hydration
- После `DOMContentLoaded` движок находит все узлы с `data-island-type`
- `IntersectionObserver` гидрирует остров только при попадании в область видимости
- После инициализации вызывается `unobserve`, чтобы избежать повторного монтирования

## Как запустить
1. Перейти в папку `lab_14.2`.
2. Открыть `index.html` в браузере (или через локальный статический сервер).
3. Взаимодействовать с островами:
- изменить количество в `counter`
- выбрать цвет в `color-picker`

## Архитектурные преимущества
- Меньше JavaScript на старте страницы
- Независимая разработка и тестирование островов
- Масштабируемая модель подключения интерактивности
- Подход близок к современным метафреймворкам, использующим partial hydration
