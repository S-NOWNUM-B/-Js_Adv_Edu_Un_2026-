# Lab 14.1: SSR и Hydration на Node.js

## Обзор
В данной лабораторной работе реализован учебный пример **Server-Side Rendering (SSR)** и последующей **Hydration** без использования фреймворков. Сервер на Node.js формирует готовый HTML с начальными данными, а клиентский скрипт подключает интерактивность после загрузки страницы.

## Цель работы
На практике изучить базовую механику SSR и hydration: как сервер подготавливает DOM и данные, а браузер «оживляет» уже отрендеренный интерфейс обработчиками событий.

## Структура файлов
- `server/index.js` — HTTP-сервер, SSR-рендер HTML, инъекция начального состояния, отдача клиентского скрипта

## Ключевые элементы реализации

### Серверный рендеринг (SSR)
- Сервер обрабатывает маршрут `/` и получает данные пользователя асинхронно
- На сервере формируется HTML с уже готовым профилем пользователя
- В страницу встраивается `window.__INITIAL_STATE__` для передачи состояния в клиент

### Клиентская гидрация
- Клиентский код загружается по маршруту `/client.js`
- После `DOMContentLoaded` читается начальное состояние и выполняется поиск целевых узлов
- Узел профиля переводится из состояния `loading` в интерактивное
- Кнопка получает обработчик переключения статуса `Offline/Online`

### Демонстрация жизненного цикла
- Сначала пользователь видит готовый контент (SSR)
- Затем появляется интерактивность (Hydration)
- Искусственная задержка помогает визуально отличить эти этапы

## Как запустить
1. Перейти в папку `lab_14.1`.
2. Запустить сервер командой:

```bash
node server/index.js
```

3. Открыть в браузере `http://localhost:3000`.

## Архитектурные преимущества
- Быстрый первый рендер страницы за счёт SSR
- Понятное разделение ответственности сервера и клиента
- Контролируемая гидрация поверх уже существующего DOM
- Простая база для перехода к SSR-фреймворкам (Next.js, Nuxt, SvelteKit)
