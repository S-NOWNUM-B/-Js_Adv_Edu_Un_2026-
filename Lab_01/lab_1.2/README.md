# Lab 1.2: Provider Pattern Implementation

## Обзор
Лабораторная работа посвящена реализации паттерна **Provider** в среде React. Данный подход используется для решения проблемы "Prop Drilling" (прокидывания пропсов через множество уровней компонентов) в приложении пользовательского дашборда.

## Технический стек
*   **Framework:** React 18+ (Functional Components)
*   **State Management:** React Context API
*   **Build Tool:** Vite

## Структура проекта в src:
*   `contexts/` — Содержит `UserContext.jsx` и провайдер, управляющий состоянием пользователя, прав доступа и темы.
*   `components/` — Иерархия вложенных компонентов (Dashboard, Header, UserMenu), демонстрирующая получение данных через кастомный хук `useUser()`.
*   `App.jsx` — Корневой компонент, оборачивающий приложение в провайдер.

## Реализованный функционал
1.  **Context Creation:** Создание контекста с обработкой ошибок (проверка использования хука вне провайдера).
2.  **Provider:** Управление глобальным состоянием (данные профиля, права `canEdit/canDelete`, настройки темы).
3.  **Consumption:** Глубоко вложенные компоненты получают доступ к данным напрямую, минуя промежуточные звенья.

## Инструкция по запуску
1.  Установите зависимости: `pnpm install` (или npm/yarn).
2.  Запустите проект в режиме разработки: `pnpm dev`.
3.  Откройте `http://localhost:5173` для просмотра работы паттерна.
