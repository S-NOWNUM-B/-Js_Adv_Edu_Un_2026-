# Lab 1.1: The Pattern Audit

## Обзор
В данной работе проведен критический анализ устаревшего кода (Legacy Code) корзины покупок на предмет соответствия принципам проектирования. Основной упор сделан на разграничение между "ад-хок" решениями, прото-паттернами и полноценными паттернами проектирования.

## Цель работы
Применить теоретические критерии из книги Эдди Османи (главы 1 и 2) для валидации программных решений. Использовать "Правило трех" (Rule of Three) и тесты на "паттерновость" (Pattern-ity Tests) для оценки кода.

## Состав лабораторной работы:
*   `lab 1.1.pdf` — Итоговый отчет, содержащий:
    *   Идентификацию и категоризацию паттернов.
    *   Оценку по «Правилу трех» (Fitness of Purpose, Usefulness, Applicability).
    *   Проверку через Pattern-ity Tests.
    *   Анализ анти-паттернов (загрязнение глобального пространства имен, сильная связанность).

## Методология
Анализ проводился путем деконструкции предложенного JavaScript-кода. В ходе аудита были выявлены недостатки архитектуры, которые превращают потенциально полезные решения в анти-паттерны, и предложены способы их рефакторинга в соответствии с современными стандартами разработки.
