# План рефакторинга

### Цель
Устранить загрязнение глобального пространства имён и жесткую зависимость в коде интенет-магазина

### Выбранные патерны
1. Namespace Pattern (паттерн Пространства имён)Ж Для создания иерархической структуры MyApp.Modulse
2. Module Pattern (паттерн модуль)Ж Для инкапсуляции логики и защиты данных через замыкание

### Этапы работы
1. Создать базовый объект пространства имён в файле namespace.js
2. Вынести настройки в отдельный модуль Config
3. Вынести вспомогательные функции в модуль Utils
4. Перенести основную логику в модуль CartCore
5. Связать все модули в главном файле main.js
