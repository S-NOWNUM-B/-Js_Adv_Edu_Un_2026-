# Документация процесса рефакторинга

### Что было изменено
1. Глобальные переменные (cart, taxRate) удалены. Данные теперь хранятся внутри модулей.
2. Вместо прямого обращения к переменным используются методы .get() и .addItem().
3. Код разделен на 5 файлов по зонам ответственности.

### Как паттерны решили проблемы
- Module Pattern: Решил проблему "Жесткой связи" (Tight Coupling). Теперь модули общаются через четкий API. Если мы решим изменить способ хранения корзины, нам не нужно переписывать весь код, только модуль Cart.
- Namespace Pattern: Решил проблему "Загрязнения имен". Мы используем только одно имя в глобальном пространстве — MyApp.

### Преимущества
- Код стал безопасным (приватность данных).
- Код стал тестируемым (каждый модуль можно проверить отдельно).
- Код соответствует принципу DRY (Don't Repeat Yourself).
