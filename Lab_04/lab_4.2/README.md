# Lab 4.2: Refactoring with Module and Namespace Patterns

## Обзор
В данной лабораторной работе выполнен рефакторинг проблемного кода с применением паттернов **Module** и **Namespace**. Цель — устранить глобальные переменные, снизить связность и сделать архитектуру расширяемой.

## Цель работы
Сформировать план рефакторинга, применить паттерны инкапсуляции и документировать изменения, демонстрируя улучшение качества кода.

## Структура файлов в lab_4.2:
*   `refactoring_plan.md` — План действий и выбранные паттерны.
*   `refactoring_documentation.md` — Описание внесенных изменений и их эффекта.
*   `refactored_modules/` — Результат рефакторинга:
    *   `namespace.js` — Инициализация пространства имен.
    *   `module1.js`, `module2.js`, `module3.js` — Модули с разделенной ответственностью.
    *   `main.js` — Точка сборки и демонстрация использования модулей.

## Методология
Работа выполнена по этапам: планирование, выделение областей ответственности, инкапсуляция данных через модули и проверка связности через публичный API.
