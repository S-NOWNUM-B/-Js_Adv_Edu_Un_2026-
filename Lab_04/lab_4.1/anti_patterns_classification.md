# Классификация антипаттернов

### Глобальные переменные
Классификация: Плохое решение (Bad Solution).
Неблагоприятная ситуация: Риск случайной перезаписи данных другими скриптами.
Как исправить: Использовать паттерн "Пространство имен" (Namespace Pattern), чтобы объединить переменные в один объект.

### Модификация прототипа Object
Классификация: Плохое решение (Bad Solution).
Неблагоприятная ситуация: Непредсказуемое поведение всех объектов в системе.
Как исправить: Вынести функции в отдельный модуль утилит.

### Дублирование кода
Классификация: Плохое решение (Bad Solution).
Неблагоприятная ситуация: Риск рассинхронизации логики при обновлении кода.
Как исправить: Централизовать логику в одной функции внутри модуля.

### Строки в setTimeout
Классификация: Плохое решение (Bad Solution).
Неблагоприятная ситуация: Открытые дыры в безопасности приложения.
Как исправить: Передавать ссылку на функцию напрямую, например: setTimeout(updateTimer, 1000).

### Жесткая связь
Классификация: Плохое решение (Bad Solution).
Неблагоприятная ситуация: Изменение одной части кода ломает несколько других.
Как исправить: Применить паттерн "Модуль" для инкапсуляции состояния и предоставления четкого API.
