## 2. Конкретные промпты

### Промпт 1:
"Что означает ошибка 'Ожидалось запятая' в объекте JavaScript после метода?"

Что получил: Объяснение правила синтаксиса — в объектных литералах методы должны быть разделены запятыми. Конкретное решение: добавить запятую после метода `get()` и перед методом `set()`.

### Промпт 2:
"Объясни паттерн Singleton и его применение в управлении конфигурацией"

Что получил: Описание паттерна Singleton как механизма создания единственного экземпляра класса. Использование статического метода `getInstance()` для контроля инстанцирования. Применение для глобального управления состоянием.

### Промпт 3:
"Как правильно реализовать Singleton в JavaScript с проверкой единственности"

Что получил: Техника с приватным свойством `_instance` и статическим методом, проверяющим наличие экземпляра перед созданием. Ленивая инициализация при первом обращении.

### Промпт 4:
"Найди ошибку в коде из-за чего прогармма не работает и объясни как исправить"

Что получил: Объяснение почему не рабоатет мой код и как его исправить чтобы всё работало более корректно

## 3. Модификации и проверки
- Исправление синтаксической ошибки (добавление запятой в объект конфигурации)
- Реализация Singleton паттерна для системы конфигурации
- Проверка единственности экземпляра при множественных обращениях
- Тестирование сохранения состояния между вызовами

## 4. Что узнал
1. Синтаксические требования JavaScript для объектных литералов
2. Как реализовать Singleton паттерн на современном JavaScript
3. Различие между глобальным состоянием и гарантированной единственностью экземпляра
4. Применение Singleton для управления конфигурацией приложения

# AI_report lab_7.2

## 1. Информация о использовании AI
- Инструмент: Copilot / GitHub, Gemini / Google AI Studio
- Дата: 26.02.2026

## 2. Конкретные промпты

### Промпт 1:
"Почему Node.js не может найти модуль './NotificationFactory' в ES6 import?"

Что получил: Объяснение требований Node.js для ES6 импортов — необходимость явного указания расширения `.js`. Включение информации о `"type": "module"` в `package.json`.

### Промпт 2:
"Объясни паттерн Factory Method и его отличие от Singleton"

Что получил: Factory Method создает семейства объектов (различные типы уведомлений), Singleton гарантирует единственный экземпляр. Разница в назначении: создание разных типов vs. контроль единственности.

### Промпт 3:
"Как реализовать Factory Method для создания различных типов уведомлений"

Что получил: Структура с фабричным методом `create(type, options)`, использующим switch для выбора нужного класса. Каждый класс имеет единый интерфейс с методом `send()`.

### Промпт 4:
"Найди ошибку в коде из-за чего прогармма не работает и объясни как исправить"

Что получил: Объяснение почему не рабоатет мой код и как его исправить чтобы всё работало более корректно

### Промпт 4:
"Какие преимущества дает Factory Method для расширяемости кода"

Что получил: Принцип открытости/закрытости (Open/Closed Principle) — код открыт для расширения (новые типы уведомлений) но закрыт для модификации клиентского кода. Легкое добавление новых типов без изменения существующего.

## 3. Модификации и проверки
- Исправление импортов (добавление расширения `.js` к имени модуля)
- Создание `package.json` с конфигурацией `"type": "module"`
- Реализация Factory Method для создания Email, SMS и Push уведомлений
- Демонстрация единого интерфейса для разных типов уведомлений
- Валидация работоспособности через запуск `node app.js`

## 4. Что узнал
1. Требования Node.js для работы с ES6 модулями и импортами
2. Как реализовать Factory Method паттерн на JavaScript
3. Преимущества абстракции типов через фабрики
4. Принцип открытости/закрытости в контексте паттернов проектирования
5. Различие между простой фабрикой и паттерном Factory Method
