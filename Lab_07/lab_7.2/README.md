# Lab 7.2: Factory Method Pattern

## Обзор
В данной лабораторной работе выполнена реализация паттерна **Factory Method** на JavaScript. Паттерн определяет интерфейс для создания объекта, но позволяет подклассам решать, какой класс инстанцировать. В данной работе Factory использется для создания различных типов уведомлений (Email, SMS, Push).

## Цель работы
Освоить реализацию Factory Method паттерна, понимать различие между простой фабрикой и паттерном Factory Method, а также применять паттерн для создания семейств взаимосвязанных объектов.

## Структура файлов в lab_7.2:
*   `NotificationFactory.js` — Фабрика для создания различных типов уведомлений с методом `create(type, options)`
*   `app.js` — Приложение, демонстрирующее использование фабрики для отправки разных типов уведомлений
*   `package.json` — Конфигурация для поддержки ES6 модулей (`"type": "module"`)
*   `notification/EmailNotification.js` — Класс для отправки email уведомлений
*   `notification/SMSNotification.js` — Класс для отправки SMS уведомлений (с ограничением 160 символов)
*   `notification/PushNotification.js` — Класс для отправки push-уведомлений

## Ключевые компоненты

### Factory Method (`NotificationFactory.js`)
- `create(type, options)` — Создает объект уведомления нужного типа
- `getSupportedTypes()` — Возвращает список поддерживаемых типов
- Использует switch для выбора конкретного класса
- Клиент не знает о конкретных реализациях

### Классы уведомлений
Все классы имеют единый интерфейс с методом `send(message)`:
- **EmailNotification** — Параметры: `to`, `subject`
- **SMSNotification** — Параметры: `to` (с ограничением 160 символов)
- **PushNotification** — Параметры: `deviceToken`, `title`

## Архитектурные преимущества

### Принцип открытости/закрытости (Open/Closed Principle)
- Код закрыт для модификации (app.js не меняется)
- Открыт для расширения (легко добавить новые типы)

### Разделение ответственности
- Каждый класс отвечает только за один тип уведомления
- Фабрика отвечает только за создание нужного типа
- Клиент работает с единым интерфейсом

## Различие между Singleton и Factory Method

| Аспект | Singleton | Factory Method |
|--------|-----------|----------------|
| Назначение | Единственный экземпляр | Создание разных объектов |
| Результат | Один объект | Множество разных объектов |
| Применение | Конфигурация, логирование | Семейства связанных типов |
| Когда использовать | Глобальное состояние | Выбор реализации во время выполнения |
