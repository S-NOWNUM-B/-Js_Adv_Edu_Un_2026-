# Lab 7.1: Singleton Pattern

## Обзор
Лабораторная работа посвящена изучению и реализации паттерна **Singleton** на JavaScript. Паттерн обеспечивает наличие единственного экземпляра класса и предоставляет глобальную точку доступа к этому экземпляру.

## Цель работы
Освоить реализацию Singleton паттерна, понимать его назначение и применение для управления глобальным состоянием приложения.

## Структура файлов в lab_7.1:
*   `singleton.js` — Реализация Singleton паттерна с методом `getInstance()` для получения единственного экземпляра
*   `config.js` — Конфигурационный объект с методами `get()`, `set()` и `getAll()` для управления состоянием
*   `test.js` — Тестирование единственности экземпляра и сохранения состояния между вызовами

## Ключевые компоненты

### Singleton паттерн
- Использование приватного статического свойства `_instance`
- Проверка существования экземпляра перед созданием нового
- Ленивая инициализация (создание только при первом обращении)
- Статический метод `getInstance()` для единственной точки доступа

### Config объект
Методы управления конфигурацией:
- `get(key)` — получить значение конфигурации
- `set(key, value)` — установить значение
- `getAll()` — получить все конфигурации

## Исправленные ошибки
- Синтаксическая ошибка в `config.js`: добавлена запятая между методами объекта

## Запуск программы
```bash
node Lab_07/lab_7.1/test.js
```

## Преимущества Singleton
- Глобальная точка доступа к единственному экземпляру
- Ленивая инициализация экземпляра
- Контролируемое управление глобальным состоянием
- Гарантия единственности

## Потенциальные проблемы
- Усложняет тестирование (сложнее мокировать)
- Может скрыть зависимости между компонентами
- Глобальное состояние может привести к побочным эффектам
