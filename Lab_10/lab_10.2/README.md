# Lab 10.2: MVVM User Profile Form

## Обзор
В подлабораторной работе реализована форма профиля пользователя по архитектуре **MVVM (Model-View-ViewModel)**. Приложение демонстрирует реактивное обновление интерфейса, валидацию полей и вычисляемые свойства без использования фреймворков.

## Цель работы
Освоить подход MVVM и научиться отделять состояние и логику валидации от слоя представления.

## Структура файлов
- `indes.html` — точка входа и подключение модулей
- `mvvm/base/ViewModel.js` — базовый класс с подписками и нотификацией
- `mvvm/view/UserProfileViewModel.js` — состояние формы, вычисляемые свойства, валидация
- `mvvm/viewmodel/UserProfileForm.js` — рендер формы и обработка пользовательских событий

## Ключевые компоненты

### Base ViewModel (`ViewModel`)
- `defineProperty(name, initialValue)` — реактивное свойство
- `$watch(name, callback)` — подписка на изменения свойства
- `notify(name, newValue, oldValue)` — уведомление подписчиков
- `$data()` — снимок текущего состояния

### UserProfileViewModel
- Поля: `firstName`, `lastName`, `email`, `age`, `errors`
- Вычисляемые свойства: `fullName`, `isValid`
- Методы: `validate()`, `save()`, `reset()`

### UserProfileForm
- Рендер формы с отображением ошибок
- Двустороннее связывание через `input`-события
- Подписка на изменения ViewModel и перерисовка интерфейса

## Исправленные ошибки
