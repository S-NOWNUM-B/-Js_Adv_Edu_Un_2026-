# Lab 10.1: MVC Todo Application

## Обзор
В подлабораторной работе реализовано Todo-приложение по архитектуре **MVC (Model-View-Controller)**. Приложение поддерживает полный базовый цикл работы с задачами: создание, редактирование, удаление и изменение статуса выполнения.

## Цель работы
На практике закрепить паттерн MVC и понять, как разделение слоев упрощает поддержку и расширение JavaScript-приложения.

## Структура файлов
- `index.html` — точка входа, стили и подключение контроллера
- `mvc/model/TodoModel.js` — модель данных и бизнес-логика Todo
- `mvc/view/TodoView.js` — рендеринг интерфейса и обработка DOM-событий
- `mvc/controller/TodoController.js` — связывание Model и View, маршрутизация действий

## Ключевые компоненты

### Model (`TodoModel`)
- Хранит массив задач
- Выполняет CRUD-операции
- Поддерживает подписку наблюдателей через `subscribe()` и уведомления через `notify()`

### View (`TodoView`)
- Рендерит список задач, статистику и форму добавления
- Делегирует действия пользователя наружу через колбэки `onAddTodo`, `onToggleTodo`, `onDeleteTodo`, `onEditTodo`
- Поддерживает inline-редактирование задачи

### Controller (`TodoController`)
- Создает экземпляры Model и View
- Связывает пользовательские действия с методами модели
- Инициализирует первую отрисовку
