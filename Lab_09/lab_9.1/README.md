# Lab 9.1: Observer Pattern (Publish/Subscribe)

## Обзор
В данной части лабораторной работы реализован поведенческий паттерн **Observer** в варианте **Publish/Subscribe** через централизованную шину событий `EventBus`.

## Цель работы
Научиться проектировать событийное взаимодействие между независимыми компонентами без жесткой связанности между издателями и подписчиками.

## Структура файлов в lab_9.1:
*   `pubsub/EventBus.js` — Шина событий с методами `subscribe`, `subscribeOnce`, `publish`, `clear`.
*   `publisher/NewsPublisher.js` — Публикатор новостей по категориям и приоритетам.
*   `subscribers/EmailNotifier.js` — Email-подписчик с подпиской на выбранные категории и поддержкой отписки.

## Ключевые компоненты

### EventBus
- Регистрация обычных и одноразовых подписчиков
- Публикация событий в нужные каналы
- Отписка через функцию, возвращаемую при `subscribe`
- Очистка подписок по событию или полностью

### NewsPublisher
- Формирование объекта статьи (категория, заголовок, приоритет, время)
- Публикация в каналы `news:<category>` и `news:all`
- Дополнительная публикация в `news:urgent` для срочных новостей

### EmailNotifier
- Подписка на несколько категорий одновременно
- Обработка новых статей и отправка уведомления
- Массовая отписка от всех активных подписок

## Архитектурные преимущества
- Слабая связанность между источником событий и получателями
- Простое расширение: можно добавлять новые типы подписчиков без изменения издателя
- Поддержка тематических каналов для гибкой маршрутизации событий
