# Lab 9.2: Command and Mediator Patterns

## Обзор
В данной части лабораторной работы реализованы поведенческие паттерны **Command** и **Mediator** для сценария управления устройствами умного дома.

## Цель работы
Освоить инкапсуляцию действий в команды с поддержкой `undo/redo` и централизованную координацию устройств через медиатор.

## Структура файлов в lab_9.2:
*   `commands/Command.js` — Базовый интерфейс команды с методами `execute()` и `undo()`.
*   `commands/CommandManager.js` — Менеджер выполнения команд, истории, `undo` и `redo`.
*   `devices/Device.js` — Базовый класс устройства с состоянием, медиатором и уведомлением об изменениях.
*   `devices/Linght.js` — Реализация устройства освещения (`Light`) с управлением питанием и яркостью.
*   `mediator/HomeMediator.js` — Медиатор для регистрации устройств, обработки изменений и запуска правил автоматизации.

## Ключевые компоненты

### Command + CommandManager
- Единый контракт команд: `execute/undo`
- Хранение истории выполненных действий
- Поддержка повторного выполнения через `redoStack`
- Ограничение глубины истории (`maxHistory`)

### Device + Light
- Базовая модель состояния устройства
- Уведомление медиатора при изменениях
- Конкретные операции освещения: включить, выключить, изменить яркость

### HomeMediator
- Централизованная регистрация и координация устройств
- Получение уведомлений об изменении состояния
- Поддержка правил автоматизации (`condition` + `action`)

## Архитектурные преимущества
- Команды упрощают откат операций и расширение новых действий
- Медиатор снижает количество прямых зависимостей между устройствами
- Логику автоматизации удобно поддерживать в одном центре управления
